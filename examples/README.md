# Example Files

This folder contains example documents you can use to test the Cycle of Service Chatbot.

## Files Included

### 1. `incomplete-recruitment-sop.txt`
**Purpose:** Practice document that needs improvement

**What to try:**
- Upload this file and ask: "Analyze this document for gaps"
- Ask: "What WOW moments should I add?"
- Ask: "Generate a complete Cycle of Service based on this"

**Expected Results:** The AI will identify:
- Missing WOW moments
- Lack of measurable actions
- Vague timing and responsibilities
- Missing customer touchpoints
- No success metrics

---

### 2. `good-employer-onboarding-sop.txt`
**Purpose:** Example of a well-structured Cycle of Service

**What to try:**
- Upload and ask: "Review this Cycle of Service"
- Ask: "What makes this a good example?"
- Ask: "Are there any areas for improvement?"

**Expected Results:** The AI will recognize:
- Clear WOW moments
- Specific measurable actions
- Complete customer journey
- Proper formatting
- May suggest minor enhancements

---

### 3. `blank-template.txt`
**Purpose:** Starting template for creating new Cycles of Service

**What to try:**
- Upload and ask: "Help me fill out this template for [your service]"
- Describe your service and let the AI guide you through completion
- Ask for suggestions at each stage

**Expected Results:** The AI will:
- Help you define each section
- Suggest appropriate WOW moments
- Recommend measurable actions
- Ensure completeness

---

## How to Use These Files

1. **Testing the Upload Feature:**
   - Drag and drop any file into the chatbot
   - Verify the file is recognized and uploaded

2. **Testing Analysis:**
   - Upload `incomplete-recruitment-sop.txt`
   - Ask the AI to identify gaps and problems

3. **Testing Generation:**
   - Upload any file or start from scratch
   - Ask the AI to create a complete Cycle of Service

4. **Testing Export:**
   - Generate a complete SOP
   - Click "Copy" to paste into Notion
   - Click "Download Word" to save as a document

---

## Creating Your Own Test Files

Feel free to create your own `.txt` files with:
- Existing process documents
- Draft SOPs
- Simple bullet-point lists
- Any service workflow you want to improve

The AI can work with various formats and levels of detail!

---

## Tips for Best Results

✅ **Do:**
- Be specific about what you want to improve
- Ask follow-up questions
- Request clarification on suggestions
- Test different types of documents

❌ **Avoid:**
- Extremely long documents (keep under 10 pages)
- Documents with complex formatting (stick to plain text)
- Multiple unrelated processes in one file
